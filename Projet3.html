Audit des Délégations AD

Projet de stage de 1ère année du 22/06/26 au 25/06/26

1 — Contexte du projet

Sur le domaine Active Directory d'Algeco (DC=int,DC=algecoscotsman,DC=com), des droits de délégation sont accordés au fil du temps sur différentes unités d'organisation (OU), sans qu'il existe de vue centralisée permettant de savoir qui possède quels droits sur quelle OU. Ce manque de visibilité complique l'audit de sécurité et le respect du principe du moindre privilège. Ce projet répond à ce besoin en automatisant la collecte et la lecture de ces délégations.

2 — Objectifs du projet

    Mon supperieur souhaitait disposer d'un état des lieux fiable des délégations en place sur l'annuaire.L'équipe IT Infrastructure : utilisait le rapport généré pour identifier d'éventuelles délégations excessives ou obsolètes et sécuriser l'annuaire.

3 — Technologies utilisées

PowerShell
Active Directory
Excel

4 — Architecture du projet

Le script interroge directement les ACL des OU de l'annuaire Active Directory plutôt que de passer par les objets ordinateurs, ce qui améliore nettement les performances. Il filtre les ACE héritées, exclut les identités BUILTIN (y compris via leur SID brut S-1-5-32-*), et ne conserve que les droits prioritaires (GenericAll, WriteDacl, WriteOwner, ExtendedRight). Un résolveur de GUID interroge à la fois le schéma et le contexte de configuration AD pour transformer les identifiants techniques en noms de droits lisibles. Le résultat est exporté dans un fichier Excel horodaté, généré via $PSScriptRoot.

5 — Travaux effectués

  • Renommage d'une variable en conflit avec $matches, variable réservée de PowerShell.
  • Ajout de l'export CSV puis migration vers un export Excel via le module ImportExcel.
  • Correction d'un bug de « unwrapping » sur les collections à un seul élément, résolu grâce à l'opérateur virgule (return ,$list).
  • Refactorisation du script pour lire directement les ACL des OU plutôt que de parcourir les objets ordinateurs, réduisant significativement le temps d'exécution.
  • Ajout de filtres : exclusion des ACE héritées, exclusion des identités BUILTIN (par nom et par SID brut), et filtrage sur les droits prioritaires (GenericAll, WriteDacl, WriteOwner, ExtendedRight).
  • Gestion sécurisée des SID orphelins pour éviter les erreurs lors de la résolution d'identité.
  • Développement d'un résolveur de GUID vers nom lisible, interrogeant le schéma et le contexte de configuration AD.
